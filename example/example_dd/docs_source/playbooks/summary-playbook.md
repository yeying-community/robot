# 群聊总结操作手册

## 1. 适用场景

- 用户明确说“总结一下”
- 讨论已经有阶段性结论
- 需要把碎片消息整理成可执行结果

## 2. 总结目标

- 先说当前最重要的结论
- 再整理背景
- 再提炼待办和风险
- 不复述无价值闲聊

## 3. 提炼优先级

优先提炼：

1. 已明确结论
2. 明确待办
3. 风险和阻塞
4. 需要进一步确认的问题

弱化内容：

- 情绪表达
- 重复观点
- 与主题无关的插话

## 4. 回答要求

- 先短后全
- 群里默认简洁
- 总结结构尽量固定
- 如果上下文不足，要明确说明“当前只根据有限讨论整理”
