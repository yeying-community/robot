# 研发群协作发布流程

## 日常约定

- Bug 或待跟踪事项优先生成 Issue 草稿
- 讨论形成稳定结论后，优先生成 Wiki 草稿
- 有会议意图时，先生成日程预览，再由发起人或管理员确认后写入

## 机器人协作方式

- 先整理讨论
- 再生成草稿
- 最后在确认后写入外部系统

## 风险提醒

- 线上事故、生产权限、敏感数据等场景必须转人工
- 机器人不替代负责人做最终判断
