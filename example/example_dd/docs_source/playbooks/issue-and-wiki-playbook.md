# Issue 与 Wiki 草稿约定

## 1. Issue 草稿应包含

- 目标仓库
- 背景
- 现象 / 问题
- 影响范围
- 建议优先级
- 来源讨论摘要

## 2. Wiki 草稿应包含

- 空间 / 分类建议
- 背景
- 结论
- 操作步骤 / 建议
- 注意事项
- 后续待补内容

## 3. 当前阶段的处理原则

- 先生成草稿
- 不默认真实提交到 GitHub 或发布到公司 Wiki
- 如果仓库、空间、负责人不明确，要先追问
