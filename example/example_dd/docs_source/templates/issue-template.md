# GitHub Issue 草稿模板

Issue 草稿默认使用以下结构：

## 标题

- 用一句话描述问题或任务

## 仓库

- 明确目标仓库，推荐使用 `owner/repo`

## 背景

- 说明问题出现的上下文

## 现象 / 问题

- 明确当前看到的现象

## 影响

- 说明影响范围或风险

## 建议优先级

- 高 / 中 / 低，并给一句理由

## 标签 / 指派建议

- 标签建议
- 负责人 / assignee 建议（如果明确）

## 待确认项

- 负责人是否明确
- 是否已有复现步骤

## 来源

- 讨论来源 / 相关文档
