# 外部工具执行策略

## 默认原则

- 先预览
- 再确认
- 后执行

## 可直接预览的内容

- GitHub Issue 草稿
- Wiki 草稿
- 钉钉日程预览

## 需要明确确认才能执行的内容

- 创建 GitHub Issue
- 创建钉钉日程
- 发布公司 Wiki（后续接入后同样适用）

默认确认人策略：

- 发起人可确认 / 取消自己的待执行动作
- 管理员可确认 / 取消当前群内的待执行动作

## 执行成功后应返回

- 做了什么
- 关键对象 ID
- 链接或后续查看方式

## 执行失败后应返回

- 错误原因
- 当前保留的草稿 / 预览
- 下一步建议
