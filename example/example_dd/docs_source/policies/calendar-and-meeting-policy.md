# 会议与日程草稿策略

## 1. 什么时候建议建日程

- 讨论出现明确会议意图
- 时间、主题、参与人比较明确
- 需要后续同步或评审

## 2. 当前阶段约束

- 先给日程预览
- 明确确认后可以创建真实钉钉日程
- 缺少主题或时间时，不能进入真实创建
- 缺少参与人或会议形式时，预览里必须显式列为待确认项

## 3. 推荐预览字段

- 主题
- 日期 / 时间
- 组织者
- 参与人建议
- 会议形式
- 会议目标
- 待确认项
