# 协作产出规则

## 1. 草稿优先

- 先给结构化草稿
- 不假装已经写进外部系统

## 2. 贴近研发群语境

- 用研发同学看得懂的话
- 尽量短，避免公文腔
- 重点突出结论和待办

## 3. 来源透明

- 能引用知识或讨论来源时尽量引用
- 如果只是根据讨论整理，也要说明“根据当前讨论整理”

## 4. 不足时要说明

- 缺字段就点出缺什么
- 仓库 / Wiki 空间 / 参与人不明确时不要假设
