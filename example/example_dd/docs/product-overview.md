# dd-bot 产品概览

## 1. 项目定义

`dd-bot` 是一个运行在钉钉研发群里的协作机器人。

它不是单纯的聊天机器人，而是一个围绕研发群协作设计的执行型助手，核心目标是把群消息转化成四类高价值结果：

- 可回答的问题
- 可跟踪的任务
- 可沉淀的文档
- 可执行的协作动作

## 2. 要解决的问题

研发群里经常发生这些情况：

- 新人不断重复问环境、权限、流程、文档
- 大家讨论了很久，但没有人整理结论
- 明明应该建日程、提 Issue、写 Wiki，却因为麻烦被搁置
- 重要知识散在聊天记录里，之后很难复用

`dd-bot` 的目标不是“更会聊”，而是：

- 让机器人能基于知识库回答常见问题
- 让机器人能把群讨论整理成结构化结果
- 让机器人能在确认后触发外部协作动作
- 让高价值讨论逐步进入可检索知识

## 3. 当前已确定的决策

- 只在 **一个研发群** 跑通 MVP
- Issue 系统接 `GitHub Issues`
- Wiki 系统接公司自研 Wiki
- 日程创建接 `钉钉日程`
- 知识库内容需要手工建设
- `yeying-rag` 是后续知识中台方向，但当前不是强依赖
- 当前主运行时选择 `OpenClaw`
- 钉钉接入选择社区 `dingtalk` channel 插件
- 模型调用采用 `Router` 主模型，`DashScope` 作为 fallback
- `docs_source/` 与产品文档会同步到 OpenClaw workspace 的 `kb/raw/`，作为本地检索源的一部分

## 4. MVP 范围

第一阶段优先支持 5 类能力：

- 知识问答
- 群消息摘要
- 待办 / 决策提炼
- GitHub Issue 草稿生成
- Wiki 草稿生成

第一阶段可以只做到：

- 日程先做预览与确认链路
- 外部写操作先从低风险、可审计的场景开始

## 5. 非目标

当前阶段明确不做：

- 全群所有消息自动回复
- 复杂长链路自主决策 Agent
- 自动把所有群消息写进知识库
- 替代项目经理或负责人做判断
- 在无确认情况下自动执行高风险写操作

## 6. 产品原则

- **可靠优先**：没有证据就不硬答
- **少打扰**：不抢答闲聊
- **草稿优先**：先产出草稿，再执行外部写入
- **日志完备**：可回溯每次判断与动作
- **OpenClaw 优先**：优先利用 OpenClaw 现成的智能体能力，不重复自建主工作流
- **演进式建设**：先在本目录跑通，再逐步接 `yeying-rag`

## 8. 当前实现状态

当前目录需要按两层来理解：

- **主线**：`OpenClaw + dingtalk 插件 + Router/DashScope + workspace 知识上下文`
- **工具层**：workspace tools / hooks / skills 负责真实写入、确认闭环与会话卫生

因此，后续开发默认优先补：

- OpenClaw workspace 内容
- OpenClaw 提示词 / 规则
- OpenClaw 工具接入

而不是继续自建机器人主脑。

## 7. 目标用户

- **新人**：需要环境、流程、文档、FAQ 支持
- **研发成员**：需要总结、Issue、Wiki、日程辅助
- **导师 / 负责人**：希望减少重复答疑与遗漏
- **维护者**：希望规则可控、结构清晰、便于扩展
